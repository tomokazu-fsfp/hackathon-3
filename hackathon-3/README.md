# チーム開発ルール（GitHub運用）

## ✅ 基本ルール
- `main` ブランチは常に動く状態を保つ（壊さない）
- 作業は必ずブランチを切って行う（例：`feature/〇〇`）
- 作業が終わったら Pull Request を作成してレビューを受ける

## ✅ ブランチ命名ルール
- `feature/〇〇`：新機能追加
- `fix/〇〇`：バグ修正
- `design/〇〇`：デザイン調整
- `docs/〇〇`：ドキュメント編集

## ✅ コミットメッセージの書き方
- `feat: ログイン画面を追加`
- `fix: ボタンの表示崩れを修正`
- `docs: READMEを追加`

## ✅ Pull Requestの流れ
1. 作業ブランチを push
2. GitHub上で Pull Request を作成
3. コメントを添えて提出
4. 他メンバーがレビュー
5. OKなら `main` にマージ

## ✅ Issueの使い方
- やること・バグ・提案はすべて Issue に登録
- 担当者を割り当てて進行管理
- 終わったら「Close」して完了

## ✅ その他
- 作業前に必ず `git pull origin main` で最新状態を取得
- 不明点は Issue やコメントで相談
